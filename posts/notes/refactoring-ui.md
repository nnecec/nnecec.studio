## 1. 从零开始

### 细节稍后处理

当开始构思一个产品时，优先考虑产品的功能实现，而不是产品的界面交互。在设计好功能之前，过早的开始设计
界面会导致很多的困扰。

在早期的设计阶段，应该尽量简单、高效的设计功能界面，例如忽略字体、阴影等细节样式，使用现实中的画笔或
者 PRD 工具制作简单的原型图，并尽量不要使用色彩来强化视觉，使用灰度设计稿会更有利于设计出层级分明的
界面。

### 不要过度设计

在这个阶段，设计产品交互、边缘情况的处理是非常困难的。不需要在一开始就设计完这些内容，而是通过短周期
的迭代工作，首先设计想要构建的功能的简单版本，然后去实现它。在实际中遇到了问题再去解决，远比提前设计
如何解决问题更加实际且高效。

### 个性化的设计

- 字体
- 色彩
- 圆角
- 语言

不要从对手的设计中参考过多的设计，那样会让人感觉你的产品只是别人的二流版本。

提前定义设计系统可以很好的解决由于无限的设计样式可选项对设计造成的困扰。不必在一开始就定义好所有东
西，只需要逐步将设计系统化，在有限的选项中匹配并通过排除法进行设计，可以有效降低决策负担，提高设计效
率。

- 字体大小
- 字体粗细
- 行高
- 颜色
- 外边距
- 内边距
- 宽度
- 高度
- 阴影
- 边框半径
- 边框宽度
- 不透明度
- ……以及任何你在设计中需要反复做出低层级决策的事项

## 2. 层级是万物之源

### 界面中的元素并不是同等层级的

### 字体大小

仅仅通过字体大小控制层级并不是一个好的设计，字体粗细和颜色深浅同样也是非常重要的。

### 字体颜色

降低透明度可以实现将字体颜色变灰从而降低其视觉权重，但透明度会使文本透出背景颜色，最好使用接近与背景
颜色的一个新的浅色颜色。

### 通过“不强调”来“强调”

当强调的手段不那么明显时，降低兄弟节点的强调层级，可以有效突出想要强调的节点。

### 标签是最后的无奈之举

- 通过标签说明会使界面显得混乱，可以利用基本认知省略标签，例如：展示人名+性别时，可以省略
  `姓名：张三 性别：男` 标签，直接使用 `张三 男` 来表示。
- 组合清晰的标签也可以使样式更简洁，例如： `房间数：3` 可以重新组合为 `3个房间`，从而使界面更清晰。
- 将标签弱化（减小字体、减轻颜色等）使标签看起来像一个单位符号。

#### 何时使用标签？

例如在产品规格页面，有大量的规格属性需要展示，且为了让用户可以直接搜索，此时仍需要使用明确的标签，例
如 iPhone 的规格页面会展示 `尺寸： 14.7cm x 7.04cm x 0.76cm`，用户通过页面查找 `尺寸` 标签，可以快速
定位到尺寸属性。

### 将视觉层级与文档层级分离

文档层级中标题从 `h1` 到 `h6` 依次从一级标题到六级标题，在默认样式中往往字体大小会依次递减。而在视觉
层级中，标题更像是一个页面标签而不是大标题，标题往往是支持性的内容而不是页面主要内容。

> 所以 tailwind 默认样式中 h1 到 h6 的标签是没有大小区分的。

### 平衡字重和对比度

加粗的字体或线条会增强层级，可以通过降低对比度来平衡视觉，例如：icon 和文字组合使用时，由于 icon 占
据的面积更大导致 icon 的视觉权重更大，此时可以通过降低 icon 的对比度来平衡视觉。

### 语义在视觉层次中不重要

语义在按钮设计中重要的组成部分，重要的按钮通过高亮引导，二级按钮应该明确但不突出，例如添加轮廓或浅色
背景，三级按钮则是在视觉中展现但不突出表现。

#### 破坏性操作

危险或者具有破坏性的操作并不一定是红色的、大的提示按钮，如果不是主要操作也可以是二级或者三级按钮。

## 3. 布局和间距

让设计变得更干净的方法首先就是为元素之间添加间距，但紧凑型的 UI 在某些数据密集型的页面仍然是有意义
的，例如：仪表盘、数据表格等场景。

### 建立间距和尺寸设计系统

- 线性刻度随着尺寸变大对设计的影响逐渐失效
- 以 16px 作为间距的基准，以指数型增长曲线建立尺寸系统
- 从系统中选择适当的尺寸填补为空白，获得更高效的设计

### 不要制造空白

不必填充完整个界面，通过收缩画布使设计更简单，例如以移动端界面作为设计基础，当设计完成后再去适配更大
的画布就会更简单高效。

> 所以 tailwind 以 mobile first 作为设计理念。

### 多列布局

当内容在宽的画布中看起来不平衡时，尝试将内容通过多列布局进行优化，而不是加宽内容的宽度。

### 网格布局被高估了

当内容或内容的某一个部分以固定宽度呈现更为合适时，例如：sidebar、登陆弹框、用户信息卡片等，以固定宽
度定义内容宽度直到画布宽度无法支持内容展示。

### 并不一定是等比例缩放

容器与容器内的元素，在不同尺寸下，并不一定是等比例缩放的。例如设计不同大小的按钮组件，小的按钮组件文
字会占据更多的空间，而大的按钮组件，文字放大到一个看起来仍比较优雅的大小，而按钮背景则可能变得更大以
更加吸引用户视线。

### 避免模糊的间距

模糊的间距会让内容变得不清晰，不同组之间的间距应该比组内元素之间的间距更大。

## 4. 设计文本

### 建立类型比例

字体大小同样应该建立设计系统，同样在更大字体大小时有更大的刻度跳跃。

避免使用 `em` 作为字体单位，而是使用 `px` 或 `rem` 作为字体单位。

### 选择好的字体

- 支持 5 个以上字重
- 为可读性进行过优化
- 选择使用量高的字体
- 从你喜欢的网站中查看字体
- 相信直觉

### 控制文本长度

将文本宽度控制在 45 到 75 个字符之间，可以有效提高可读性。

### 通过 baseline 对齐

### 行高

行高随内容宽度变大而变大，随字体变大反而可能需要变小。

### 不是所有的链接都需要颜色

可以仅在鼠标 hover 时显示下划线或颜色。

### 对齐方式

对于 LTR 的语言，左对齐更有利于阅读。长文本应该使用左对齐，表格中的数字右对齐更有利于小数点的阅读。

### 高效使用 letter-spacing

粗体大标题的场景下，通过减小 letter-spacing 提高可读性。全大写字母的文字情况下，通过增加
letter-spacing 提高可读性。

## 5. 使用颜色

使用 HSL 定义颜色

### 多种颜色

- 灰色：文字、背景、表单控件、布局背景
- 主色（primary color）：导航、按钮、主题色
- 强调色：语义 例如：成功、错误、警告

### 使用颜色来定义色板

- 首先定义基准颜色
- 调整基准颜色的深浅获得较深的和较浅的颜色
- 填补基准颜色到深色和浅色之间的空白
- 对于灰色来说，基准颜色不那么重要，只需要从一端到另一端定义颜色即可。

> tailwind 的 color 配置就是通过这种方式来定义的，首先定义 500 基准颜色，然后通过调整 500 的颜色深浅
> 获得 100 和 900 的颜色。再依次填充 200、300、400、600、700、800 的颜色。

### 亮度 饱和度 色相

### 灰色不一定是灰色的

灰色是低饱和度的，但可以调整一些色调让灰色看来起来更暖或更冷（色温）。

> tailwind 的灰色组颜色其实就是有色温的灰色。

### 可访问性

WCAG 建议小于 18px 的字体 [对比度](https://webaim.org/resources/contrastchecker/) 至少是 4.5:1，大于
18px 的字体对比度至少是 3:1。

在深色背景上使用浅色文字时，对比度应大于 4.5:1。而更好的方式是使用浅色背景和深色文字，同时具备了色彩
和可阅读性。

### 调整色调

降低文字的层级，不只可以通过缩小字体大小，降低饱和度。例如在深色背景下，调整文字的色调，使其看起来更
接近背景颜色同样可以降低文字的层级。

### 不要只依赖颜色

考虑到色盲症患者，不要只依赖颜色来区分内容。加入 icon、文字，并调整色块的亮度都可以起到区分颜色的作
用。

## 6. 创建深度

通过模拟光源，来模拟现实世界中不同深度元素的视觉效果。例如阴影越长，元素看起来越高。

### 建立高度系统

定义最小阴影和最大阴影，并填充中间的空白来建立高度系统。

在交互中加入阴影的互动，例如在鼠标 hover 时增加阴影，在点击按钮时减少阴影，都起到了增强交互的作用。

### 物体可能有两个或以上的阴影

一个阴影模拟环境光，范围更大也更柔和，另一个阴影模拟环境光源无法影响的紧凑阴影，范围更小也更紧凑。当
物体越来越高时，紧凑阴影会因为环境光源的介入而变淡并消失。

### 有深度的平面设计

- 浅色元素会感觉更近，深色元素会感觉更远
- 实心没有模糊的阴影可以增加一点深度的效果
- 重叠元素，并为上层元素增加高度

## 6. 使用图片

- 使用高质量、好看的图片

丰富的图片颜色会导致图片上方的文字难以阅读，可以通过

- 图片增加蒙版
- 降低图片对比度
- 为图片着色
- 为文字增加阴影

的方式来提升可阅读性。

### 控制图片大小

- 不要放大到图片模糊
- 如果图片太小可以将图片放到轮廓图形中
- 不要缩小屏幕截图，会使截图难以阅读。超出范围或者放大局部都是更好的选择。如果真的需要放置缩小的截
  图，可以简化界面例如删除截图中的小文字。

### 当心不规则的图片

用户上传的图片尺寸是不规则的，处理好不规则图片对布局可能的影响。同时图片边缘的颜色可能和背景颜色接
近，可以考虑为图片增加边框或阴影来划分图片和背景。

## 7. 增强默认样式

- 调整 `ul` `ol` `blockquote` 等标签的默认样式
- 调整 `a` 标签的默认样式
- 调整 `form` 中 `checkbox` `radio` 的默认样式
- 为容器、背景增加边框颜色使界面不那么平淡
- 装饰背景，为布局模块增加纯色或者渐变的背景颜色
- 添加简单图形，例如波纹、波点等

### 不要忽略空状态

设计数据空状态的界面时，如果数据依赖用户输入，那么在提示用户当前界面无数据的情况时，同时提供新增数据
的按钮引导用户添加数据。

并且可以隐藏依赖数据的交互模块如筛选器、表头等，使界面更加简洁。

### 减少使用 border

过多的线条会使界面显得凌乱

- 可以为不同的块设置不同颜色深浅的背景颜色达到区分模块的目的
- 可以增加边距来区分不同的模块

### 想想容器内可以有哪些优化

- 下拉菜单不是千篇一律的自上而下排布，而是分类为多列或多组，并设置不同的背景色、字体颜色等
- 将表格相似的内容整合，而不是平铺
- 表单单选框可以是选项卡的样式
